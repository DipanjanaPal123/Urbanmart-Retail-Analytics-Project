# Power BI Dashboard — Build Guide

Power BI Desktop (free) is a Windows app, so the `.pbix` has to be built there —
but this guide gets you a polished, resume-worthy dashboard in ~20 minutes
using files this project already generated for you.

## 1. Get Power BI Desktop
Free download: https://www.microsoft.com/en-us/power-platform/products/power-bi/downloads

## 2. Load the data
`Home > Get Data > Text/CSV`, load these files (generated by `python/analysis.py`):
- `output/fact_sales_full.csv` — main fact table
- `output/rfm_segments.csv` — customer segments
- `output/monthly_sales.csv` — pre-aggregated trend (optional, DAX can do this too)

Or connect directly to `data/urbanmart.db` via an ODBC/SQLite connector if you
want to show a live SQL → Power BI pipeline (extra credit for the interview).

## 3. Build the data model
`Model` view: relate `fact_sales_full[customer_id]` → `rfm_segments[customer_id]`
(one-to-one). Then add the `Calendar` table from `powerbi/DAX_measures.txt` and
relate `Calendar[Date]` → `fact_sales_full[order_date]` (one-to-many).

## 4. Add the measures
Copy every measure from `powerbi/DAX_measures.txt` into
`Modeling > New Measure`.

## 5. Build 3 pages

**Page 1 — Executive Overview**
- 4 KPI cards: Total Revenue, Total Profit, Profit Margin %, Avg Order Value
- Line chart: Revenue by month (`Calendar[YearMonth]` on axis, `[Total Revenue]`)
- Bar chart: Revenue by Category
- Map or bar chart: Revenue by Region
- Slicers: Year, Category, Region

**Page 2 — Customer Segmentation (RFM)**
- Donut chart: customer count by `segment` (Champions / Loyal / Potential / At Risk)
- Table: segment × Total Revenue × Avg Order Value × Active Customers
- Scatter chart: Recency (x) vs Monetary (y), size = Frequency, color = segment

**Page 3 — Forecast & Trends**
- Line chart with the built-in Power BI forecast (Analytics pane > Forecast),
  or plot `output/forecast.csv` as a second series against actuals
- YoY Growth % as a KPI visual with target line

## 6. Polish (this is what makes it "stand out")
- Consistent color theme (View > Themes) — pick 1 accent color, use it everywhere
- Add a title, your name, and "Data: Jan 2022 – Dec 2024" subtitle
- Add tooltips with extra context on hover
- Publish to Power BI Service (free account) for a shareable web link:
  `File > Publish > Publish to Power BI` → copy the link into your README/resume

## 7. Export for GitHub
Power BI files can't render on GitHub directly. Do both:
- Commit the `.pbix` file itself (recruiters who have Power BI can open it)
- Take 3-4 screenshots of each page → save to `powerbi/screenshots/` →
  embed them in the main README so the dashboard is visible without opening anything
- If you publish to Power BI Service, add that live link to your README too —
  this is the single highest-impact addition for a resume project
